Підсторінка 4

План реалізації проєкту

Готовий план, який можна показати на захисті та використати як дорожню карту.

Етапи виконання

Етап Що робимо Інструменти Термін
1. Дослідження Збір фактів про ШІ, формування структури теми. Google, підручники, статті 1 день
2. Дизайн Палітра, шрифти, макет сторінок, UX-навігація. HTML, CSS 1 день
3. Розробка Верстка підсторінок, JS (меню, тема, анімації). HTML, CSS, JS 2 дні
4. Тестування Перевірка адаптивності й валідності структури. Локальний сервер, браузер 1 день
5. Публікація Завантаження на GitHub і запуск GitHub Pages. GitHub 30 хв

Чекліст перед публікацією

  • Усі сторінки мають працюючу навігацію.
  • Є адаптивність для телефона.
  • Вказані автори проєкту.
  • Файли лежать у корені репозиторію.

Як захостити на GitHub Pages

  • Створіть репозиторій і завантажте всі файли.
  • Settings → Pages → Deploy from branch.
  • Branch: main, Folder: /root.
  • Отримайте публічний URL і вставте у Google Sites (Embed URL).
